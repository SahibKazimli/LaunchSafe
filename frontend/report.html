HOW LAUNCHSAFE SCORES RISK

LaunchSafe combines industry-standard security scoring with exploitability and business-context analysis, so findings are not just detected, but meaningfully prioritized.

CVSS-based severity CVSS

Uses CVSS-style scoring to assess severity based on exploitability and technical impact.

OWASP-style risk assessment OWASP

Uses OWASP-style reasoning to evaluate how likely an issue is and how much it matters in real applications.

Threat and business context Context

Adjusts priority using real-world context like internet exposure, sensitive data, exploit signals, and customer or compliance impact.

Severity levels
Criticaleasy to exploit and high-impact, likely to cause major compromise
Highserious weakness with significant security impact
Mediumreal issue, but requires more conditions or has limited impact
Lowlower-risk issue, defense-in-depth, or limited practical exposure

THIS SCAN'S MATH

Each finding contributes CVSS base × exposure to a total risk number. CVSS comes from the agent (CVSS v3.1 bands). Exposure is the deployment context of the file the bug lives in — production code costs full points, test fixtures barely count.

1. CVSS base CVSS

Critical 9.0–10, High 7.0–8.9, Medium 4.0–6.9, Low 0.1–3.9. Picked per finding.

2. Exposure multiplier Context

production ×1.0 · internal ×0.6 · test ×0.15 · example ×0.05 · doc ×0.03.

3. Headline grade

score = clamp(100 − 2 × Σ contribution).
